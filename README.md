# 📦 docvi – Neovim‑контейнер с полной автоматизацией

Этот репозиторий содержит **полностью готовый Docker‑образ**, в котором запущен **Neovim** с моей персональной конфигурацией. 
Скрипты `docvi_build.sh` и `docvi_setup.sh` позволяют:

* автоматически клонировать конфигурацию Neovim,
* собрать образ, 
* запустить контейнер в двух режимах (root и user),
* пользоваться кэшем Neovim (пакеты, плагины, LSP) между запусками.

---

## 📂 Структура репозитория (в рабочем виде)

```
.
├── Dockerfile                # Описание образа (Debian‑slim + neovim + утилиты)
├── docvi_build.sh           # Скачивает конфиг, собирает образ и создаёт ссылку docvi
├── docvi_setup.sh          # Обёртка для запуска контейнера (root / user / проверочный)
├── data/
│   ├── localU/              # Кеш Neovim в режиме обычного пользователя
│   ├── localR/              # Кеш Neovim в режиме root
│   └── neovim-conf/         # Склонированная папка с конфигом (см. ниже)
├── README.md                # Этот файл
└── (другие файлы проекта, .gitignore, лицензия и т.п.)
```

### `data/neovim-conf/` – конфигурация Neovim 

* `init.lua` – точка входа, инициализирует `lazy.nvim` и подключает модули.
* `lua/core/` – общие параметры (опции, keymaps, lazy‑настройка).
* `lua/plugins/` – отдельные файлы с конфигурацией каждого плагина (Telescope, Treesitter, LSP, автодополнение и т.д.).
* `lazy-lock.json` – зафиксированные версии плагинов.

## 🛠️ Как начать работать

### 1. Сборка образа и подготовка окружения

Выполните один раз (или каждый раз после изменения `Dockerfile`/конфига):

```bash
./docvi_build.sh
```

Что делает скрипт:

1. **Создаёт каталоги** `./data/localU` и `./data/localR` – места, где Neovim будет хранить кеш (пакеты, LSP‑серверы, swap‑файлы и пр.).
2. **Клонирует** репозиторий `NeovimConfig` в `./data/neovim-conf` (если он ещё не существует). Вы можете заменить его своим форком, просто отредактировав URL в скрипте.
3. **Собирает Docker‑образ** с тегом `docvi`.
4. **Создаёт симлинк** `/usr/local/bin/docvi` → `docvi_setup.sh`, чтобы запускать `docvi` отовсюду

После выполнения вы увидите сообщения вроде:

```
OK: Директория ./data/localU успешно создана
OK: Конфигурация neovim успешно загружена
...
OK: docvi готов к работе (/usr/local/bin)
```

### 2. Запуск Neovim в контейнере

```bash
docvi <файл_или_директорию>
```

В зависимости от текущих прав скрипт выбирает режим:

| Права пользователя | Что происходит |
|-------------------|-----------------|
| **root** (`id -u = 0`) | Запускается контейнер в *root‑режиме* (`docviRoot`). Внутри контейнера `HOME=/root`, кеш монтируется из `./data/localR`. |
| **обычный пользователь** | Запускается *user‑режим* (`docviUser`). `HOME` переопределяется на `/tmp`, кеш берётся из `./data/localU`. Если файл, указанный в аргументе, недоступен на запись, скрипт завершится с ошибкой. |

#### Примеры

```bash
# Открыть файл в текущем каталоге
 docvi main.py

# Открыть директорию (получите дерево файлов в Telescope)
 docvi ./src

# Показать версию neovim внутри контейнера
 docvi --version
```

## 🧩 Как работает кеш

- **`./data/localU`** – используется, когда Neovim запускается от обычного пользователя.
- **`./data/localR`** – аналогичный каталог для запуска от root.

Эти каталоги монтируются в контейнер как `/root/.local/` (root‑режим) или `/tmp/.local/` (user‑режим). 

Благодаря этому вам не нужно каждый раз переустанавливать плагины, LSP‑серверы и прочие зависимости – они сохраняются между запусками контейнера.
